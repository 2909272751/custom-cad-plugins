# OpeningOutlinePlugin

AutoCAD 2022 开洞轮廓生成插件。

这个插件用于把图纸里的洞口边界自动生成闭合 `LWPOLYLINE`。插件本身不统计面积，生成闭合线后，你可以继续用已有面积统计工具统计输出图层。

## 适合解决什么问题

在装配式算量中，洞口面积经常需要先把洞口边界描成闭合多段线。手动用 `PL` 描边很慢，遇到柱、墙柱、剪力墙遮挡时还容易走错。

`OpeningOutlinePlugin` 的目标是：

- 根据开洞符号图层自动识别洞口；
- 让用户修正识别结果；
- 根据用户点选的边界样本图层提取洞口边界线；
- 自动生成闭合洞口轮廓；
- 输出到指定图层，方便后续面积统计。

## 命令

| 命令 | 作用 |
| --- | --- |
| `DKTRACE` | 识别开洞符号并生成洞口闭合轮廓 |
| `DKLOG` | 显示诊断日志路径 |

## 安装 / 加载

在 AutoCAD 命令行输入：

```text
NETLOAD
```

选择 DLL：

```text
C:\实用软件开发\OpeningOutlinePlugin\dist\OpeningOutlinePlugin-v0.1.0-autocad2022.dll
```

加载后输入：

```text
DKTRACE
```

## 新手操作流程

### 1. 框选处理区域

命令行提示：

```text
框选处理区域，里面要包含绿色开洞符号:
```

建议第一次只框选一个或几个洞口附近，不要直接框整层。

### 2. 点选一个开洞符号

命令行提示：

```text
点选一个绿色开洞符号，用来读取图层:
```

你只需要点一个绿色开洞符号对象。插件会读取这个对象所在图层，并在刚才框选区域内自动识别同图层的开洞符号。

### 3. 修正开洞符号识别结果

识别到的开洞符号会被高亮，命令行会提示：

```text
修正选择 [Done/Add/Remove] <Done>:
```

含义：

- `Done`：识别结果正确，进入下一步
- `Add`：增加漏识别的开洞符号
- `Remove`：剔除误识别的开洞符号

如果识别没问题，直接回车即可。

如果少识别了：

```text
Add
```

然后点选漏掉的开洞符号。

如果多识别了：

```text
Remove
```

然后点选误识别对象。

### 4. 输入容差

命令行提示：

```text
端点吸附/断口容差，单位按当前图纸 <20>:
```

一般直接回车使用默认值 `20`。

如果洞口边线有小断口，可以试：

```text
30
50
```

### 5. 输入输出图层

命令行提示：

```text
生成轮廓图层 <DK_OPENING_OUTLINE>:
```

一般直接回车。成功生成的闭合轮廓会放到：

```text
DK_OPENING_OUTLINE
```

### 6. 连续点选边界样本图层

命令行提示：

```text
点选边界图层样本对象继续添加，或按 Space/Enter 完成 [Remove]:
```

这一步不是让你一根根选线，而是让你点“图层样本”。

比如可以点：

- 洞口边线
- 梁边线
- 墙边线
- 柱边线
- 墙柱边线
- 剪力墙边线
- 柱或墙柱的填充对象

每点一个对象，插件会读取这个对象所在图层，并立刻高亮框选区域内这个图层的所有对象。

如果高亮对象是你希望参与洞口围合的，就继续点下一个样本图层。

如果误选了图层：

```text
R
```

或输入：

```text
Remove
```

然后点一个该图层对象，插件会移除这个图层并刷新高亮。

选择完成后按：

```text
Space
```

或：

```text
Enter
```

### 7. 自动生成洞口轮廓

插件会根据开洞符号中心点，在候选边界线网中寻找对应的闭合环，并生成闭合 `LWPOLYLINE`。

成功结果会放到输出图层。

失败位置会在：

```text
DK_OPENING_FAILED
```

图层画红圈并标注原因。

## 边界图层怎么选

优先选择真正参与洞口围合的图层：

- 洞口边线
- 梁/墙边线
- 柱边线
- 墙柱边线
- 剪力墙边线
- 柱或墙柱填充边界

尽量不要选择：

- 开洞符号图层
- 轴线
- 尺寸标注
- 文字
- 楼梯踏步线
- 无关填充
- 与洞口围合无关的内部线

简单判断：你手动用 `PL` 描这个洞口时会沿着哪些线走，就把这些线所在图层加入边界图层。

## 支持对象

当前支持：

- `LINE`
- `LWPOLYLINE` 直线段
- `LWPOLYLINE` 弧线段
- `ARC`
- `HATCH` 边界循环

暂未重点处理：

- 块内部线条
- 天正自定义对象内部几何
- 样条曲线
- 椭圆
- 三维非平面对象

## 失败或结果不对时

输入：

```text
DKLOG
```

查看日志路径。默认是：

```text
%TEMP%\DKTRACE.log
```

日志里重点看：

- `Symbol layer`：开洞符号图层
- `Final symbol count`：最终识别到的开洞符号数量
- `Boundary layers`：参与围合的边界图层
- `Boundary entities from area by layer`：参与计算的边界对象数量
- `Raw segment count`：读取到的原始线段数量
- `Split segment count`：求交拆分后的线段数量
- `Faces`：识别到的闭合环数量
- `边界线不足` / `未找到闭合环`：失败原因

如果结果不对，把日志发给开发者，通常能判断是图层选少、图层选多、线断开、对象类型不支持，还是候选闭合环判断需要继续优化。
